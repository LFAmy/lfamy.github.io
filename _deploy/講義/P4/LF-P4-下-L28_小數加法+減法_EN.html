二、Core Knowledge （15 min） ＋ Worked Examples
⚠️ 致命Trap：Decimal點對齊 vs 右對齊
整數加減用「右對齊」（個位對個位），但Decimal加減要用「Decimal點對齊」！
❌ 錯：0.5 同 0.25 右對齊 → 5對5、0對2 → 0.5+0.25=0.30（錯！）
✅ 對：0.50 + 0.25 → Decimal點對齊 → 0.75（正確！）
知識點一：DecimalAddition 🔴 SSPA 必考
① Step 1 — Decimal點對齊：兩個數的Decimal點要喺同一條垂直線上
② Step 2 — 補零：Decimal位唔夠→補0。例：0.5 + 0.25 → 0.50 + 0.25
③ Step 3 — 如常Addition：由右至左逐位加，跟整數Addition一樣
④ Step 4 — Decimal點落返：答案的Decimal點 = 同對齊位置落返
例1
0.38 + 0.7 = ？（補0：0.38 + 0.70 = 1.08）
例2
12.5 + 8.76 = ？
知識點二：DecimalSubtraction 🔴 SSPA 必考
① 同Addition一樣：Decimal點對齊 + 補零
② 借位同整數Subtraction一樣：唔夠減就向前借1。借1 = 借嗰位嘅10個單位
③ 例：0.82 - 0.56 → 百分位：2唔夠減6 → 借位：12-6=6 → 十分位：7-5=2 → 0.26
④ 整數減Decimal：整數後面加 .0（或 .00）再減
例3
0.73 - 0.48 = ？（借位：百分位 3-8 唔夠→向十分位借→13-8=5；十分位 6-4=2 → 0.25）
例4
2 - 0.65 = ？（2 = 2.00 → 2.00 - 0.65 = 1.35）
知識點三：Decimal加減混合 + 驗算 🟡 SSPA
① 由左至右：同整數一樣，冇括號就由左至右逐個計
② 驗算方法：Addition用Subtraction驗（和 - 加數 = 另一加數）。Subtraction用Addition驗（差 + 減數 = 被減數）
③ Estimation先驗：計之前先Estimation→計完睇答案係咪合理
例5
3.5 + 2.78 - 4.1 = ？（補零→ 3.50+2.78=6.28 → 6.28-4.10=2.18）

#Common Error（❌ Trap）Correct Approach（✅）
1右對齊（同整數）：0.5+0.25→ 5對5 錯位Decimal點對齊！0.50+0.25。Decimal點在垂直線上
2冇補零：0.5+0.25 計成 0.5+0.25=0.30（❌）補0：0.50+0.25。十分位5+2=7→0.75
3Decimal點忘記落返：答案 075 冇點答案的Decimal點要同對齊位置垂直落返
4借位時忘記減1：0.82-0.56，借位後十分位仍是8借位=借嗰位減1。8→7（借咗1俾百分位）
5整數減Decimal：2-0.65=1.65?（❌）2=2.00→2.00-0.65=1.35
6加減混合次序錯由左至右逐個計。或分組加減後再合併
7答案末位零冇刪：3.50 寫成 3.5（可以）但最好不要P4可保留末位零以清晰表示準確度。3.5=3.50
🧠 Mnemonic：「Decimal加減唔同整，Decimal點對齊先開始。位數唔同補個零，由右至左逐位計。借位同整數一樣做，Decimal點直落唔會錯。」
七、4-Step Problem Solving
1
對齊Decimal點
兩個數Decimal點喺同一垂直線。這是唯一正確的寫法！
2
補零
Decimal位數不同→在短的數後面補0。0.5 + 0.25 → 0.50 + 0.25
3
逐位加減
由右至左（百分位→十分位→個位...）逐位計算。借位照做
4
落Decimal點
答案的Decimal點 = 喺對齊位置直落。驗算：Estimation覆核
