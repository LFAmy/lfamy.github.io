1 建立 Google Sheets CRM 後端

1.1 建立新試算表

  1. 前往 sheets.google.com
  2. 點「+ 空白」建立新試算表
  3. 改名為「霖楓 CRM」

1.2 開啟 Apps Script 編輯器

  1. 試算表內：選單「擴充功能」→「Apps Script」
  2. 會開新分頁，標題顯示「程式碼.gs」
  3. 刪除編輯器內所有預設代碼

1.3 貼上 CRM 後端代碼

  1. 在專案資料夾打開：_operations/apps_script_crm.js
  2. 複製全部內容
  3. 貼入 Apps Script 編輯器
  4. 點「儲存」(💾 icon)

1.4 部署為網路應用程式

  1. 右上角點「部署」→「新增部署」
  2. 類型：選「網路應用程式」
  3. 執行身分：「我」
  4. 存取權：「任何人」
  5. 點「部署」
  6. 彈出授權視窗 → 點「允許」
⚠️ 重要：複製出現的 Web App URL！格式類似：
https://script.google.com/macros/s/ABCDEFGHIJKLMNOPQRSTUVWXYZ/exec
呢個 URL 係成個系統的後端，track.html 同 crm.html 都要用。

2 把 Apps Script URL 填入系統

需要改 2 個檔案，每個改 1 行。

2.1 更新 track.html

  1. 打開 docs/track.html
  2. 搜尋（Ctrl+F）：YOUR_GOOGLE_APPS_SCRIPT_URL
  3. 替換為 Step 1 複製的 Apps Script URL
  4. 儲存

2.2 更新 crm.html

  1. 打開 docs/crm.html
  2. 搜尋（Ctrl+F）：YOUR_GOOGLE_APPS_SCRIPT_URL
  3. 替換為 Step 1 複製的 Apps Script URL
  4. 儲存

3 設定 FB Pixel

3.1 取得 Pixel ID

  1. 前往 FB 事件管理工具
  2. 選擇你的 Pixel（如未有，點「建立」）
  3. 複製你的 Pixel ID（16 位數字）

3.2 更新所有檔案

在以下檔案中搜尋 XXXXXXXXXXXXXXXX 並替換為你的 Pixel ID：

檔案用途
index.html根首頁 FB Pixel
docs/index.html主站首頁 FB Pixel
docs/track.htmlUTM 追蹤頁 FB Pixel + Lead 事件

4 設定每日自動跟進提醒

  1. 回到 Apps Script 編輯器
  2. 左側點「觸發條件」icon（⏰ 鬧鐘）
  3. 點右下「新增觸發條件」
  4. 設定如下：
    • 執行的功能：checkFollowUps
    • 部署：Head
    • 事件來源：時間驅動
    • 時間型別：日計時器
    • 時間段：上午 9 點至 10 點
  5. 點「儲存」
每日早上 9 點自動掃描 CRM，如果有超過 1 日未跟進的新 lead → 自動 send email 提醒你。

5 更新 FB 廣告連結

5.1 使用廣告連結產生器（推薦）

  1. 開瀏覽器：https://www.lfacademyhk.com/docs/ad-link-generator.html
  2. 選擇廣告平台、輸入系列名稱、廣告版本、目標年級
  3. 點「產生連結」→「複製連結」
  4. 貼入 FB 廣告的「網站 URL」欄位

5.2 手動產生 UTM 連結格式

https://www.lfacademyhk.com/track.html?utm_source=fb&utm_medium=cpc&utm_campaign=系列名&utm_content=版本&grade=年級

驗證清單

  1. 用手機㩒 track.html UTM 連結 → 確認自動跳轉 WhatsApp
  2. WhatsApp 訊息帶有來源標記（例：[P5-呈分試急補-v1]）
  3. 打開 Google Sheets → 確認 Leads sheet 有新 row
  4. 開 /docs/crm.html → 確認顯示 lead 數據
  5. 開 /docs/ad-link-generator.html → 確認可產生連結
  6. FB 事件管理工具 → 確認收到 Lead 事件
  7. Apps Script 觸發條件 → 確認 checkFollowUps 已設定

疑難排解

問題解決方法
track.html 無法記錄到 Sheets 確認 SHEETS_WEBHOOK URL 正確，Apps Script 部署為「任何人」可存取
crm.html 顯示「無法連接」 確認 API_URL 正確，Apps Script 的 doGet() 已部署
WhatsApp 訊息沒有來源標記 確認 UTM 參數在 track.html URL 中有傳遞
FB Pixel 事件無數據 安裝 FB Pixel Helper Chrome 擴展檢查
Apps Script 權限錯誤 Apps Script 編輯器 → 左側「專案設定」→ 勾選「將 Google 帳戶的資料顯示...」

日常使用流程

銷售每日 SOP

  1. 早上開 /docs/crm.html 睇新 leads
  2. Check email（Apps Script 跟進提醒）
  3. WhatsApp 跟進紅色 overdue leads
  4. 每次傾完 → 手機 Google Sheets → 更新 Status 欄位為：
    • Contacted（已聯繫）
    • Trial（試堂）
    • Enrolled（報名）
    • Lost（流失+原因）
  5. 收工前睇 CRM 儀表板 → 確認轉換率

每週市場總監 Review

  1. CRM 儀表板：總 leads / 轉換率 / 各廣告系列成效
  2. Google Sheets → 按 Campaign 排序 → 計每個廣告 ROI
  3. FB 廣告管理員 → 睇 CPM/CTR/CPA 趨勢
  4. 決定：加 budget 邊個廣告、停邊個廣告
