## 🟡 等級 2：重要問題（本週期應修復）

### 2.1 SVG 圖形覆蓋率不足
| 級別 | 含SVG比例 | 缺圖類型 |
|------|----------|---------|
| P3 | 7/40 (18%) | 立體圖形、對稱、棒形圖、方向位置 |
| P4 | 25/40 (63%) | 複合圖形面積、對稱、棒形圖 |
| P5 | 17/40 (43%) | 梯形/多邊形面積、排水法、複合立體 |
| P6 | 23/40 (58%) | 圓形圖、折線圖、排水法 |

**影響**：大量幾何題目只有文字描述，學生無法視覺化理解。

### 2.2 作答空間高度不統一
- P3-P4：全部固定 80px
- P5：混亂（44px / 72px / 75px / 90px / 100px）
- P6：混亂（28px / 30px / 44px / 72px / 80px）

**建議標準**：基礎題 44px、進階題 65px、挑戰題 90px、應用題 110px。

### 2.3 命名不統一
- P6 有 5 個 `學生版講義_P6_上_Lxx_xxx_v6.html`（非標準命名）
- P5 有 `LF-P5-上-L06-綜合-面積-多位數-估算-v1.html`（含 v1 標記、連字符混用）
- SSPA 模擬卷散落各處，缺乏統一命名規範

### 2.4 難度標記系統不一致
- P3-P4：用 emoji（🌱🌿🌳🏔️）
- P5：混合 emoji + 陷阱編號 T1-T10
- P6：用文字（基礎/進階/挑戰/極限）
- P5 部分文件使用 HTML 實體編碼（`&#x1F331;`）而非直接 Unicode，影響可讀性

## 🟢 等級 3：系統架構級問題（Phase 6 處理）

### 3.1 無版本控制
**最高風險** — 80+ 文件被直接編輯，無法回滾、無法追蹤變更、無法協作。

### 3.2 英文翻譯 Pipeline 未建立
- P5 EN：僅 L01 完成（1/40）
- P6 EN：僅 L01 完成（1/40）
- 剩餘 78 份需手動翻譯，每份約 30-45 分鐘 = **~45 工時**

### 3.3 工具鏈缺乏統一入口
- `_tools/` 有 40+ 腳本
- `_operations/` 有 20+ 腳本
- 無 CLI 統一入口、無腳本文檔、無錯誤處理

### 3.4 P3-P4 急需 v6 升級
P3-P4 完全沒用 v6 模板，缺少：
- CSS 分數系統 (`.f`, `.fd`, `.fi`)
- 陷阱對比表 (`.tc-w`, `.tc-r`)
- 知識點區塊 (`.kp`)
- 口訣區塊 (`.mn`)
- 分層難度標記 (`d1/d2/d3/d4`)
- 作答區格線 (`qw`)

### 3.5 PDF 與 HTML 不同步
所有 PDF 為靜態快照，HTML 被更新後 PDF 未重建。重建腳本存在但需要手動觸發。
